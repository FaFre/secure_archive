# secure_archive

## Workspace
- This repo is a Dart pub workspace rooted at `pubspec.yaml`; workspace members are `apps/secure_archive_app` and `packages/secure_archive`.
- Run dependency install from the repo root with `dart pub get`.
- Repo-level scripted commands come from Melos in the root `pubspec.yaml`; run them as `dart run melos <script>`.

## Package Boundaries
- `apps/secure_archive_app` is the Flutter app. The main UI is concentrated in `apps/secure_archive_app/lib/main.dart`.
- `packages/secure_archive` is the reusable Dart library. Public exports are in `packages/secure_archive/lib/secure_archive.dart`; the pack/unpack flow is implemented in `packages/secure_archive/lib/src/domain/archive.dart`.
- The app depends on the local library via a path dependency, so shared archive behavior should usually be changed in `packages/secure_archive`, not duplicated in the app.

## Verification
- Analyze the library from `packages/secure_archive` with `dart analyze`.
- Analyze the app from `apps/secure_archive_app` with `flutter analyze`.
- Run library tests from `packages/secure_archive` with `dart test`.
- Run a single library test from `packages/secure_archive` with `dart test test/split_test.dart` or `dart test test/archive_flow_test.dart`.
- Library tests use `Directory.current` to load `test/fixtures` and `test/archives`, so running them from the package directory matters.

## Generated Code
- `packages/secure_archive/lib/src/data/models/*.g.dart` is generated by `json_serializable`.
- After editing `@JsonSerializable` models or their converters, regenerate with `dart run melos build` from the repo root, or `dart run build_runner build --delete-conflicting-outputs` in `packages/secure_archive`.

## Repo Quirks
- The Flutter app is desktop-oriented: `window_manager` setup runs only on Windows/Linux/macOS in `lib/main.dart`.
- The app and current tests use `Argon2Params.memoryConstrained()` rather than the much heavier `recommended()` profile.
